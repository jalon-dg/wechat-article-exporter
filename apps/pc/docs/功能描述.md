# 微信文章导出器 - PC 桌面端

## 1. 产品定位

一款独立的 Windows/Mac 桌面客户端，让用户可以在本地完成公众号文章的一键采集、下载和电子书导出，无需依赖云端服务。

## 2. 核心功能

### 2.1 公众号登录
- 用户通过微信扫二维码登录自己的公众号
- 登录后会保存 cookie/token，支持自动续期
- 支持多账号切换

### 2.2 公众号搜索与采集
- 输入公众号名称进行搜索
- 查看公众号基本信息（头像、简介、粉丝数、文章数）
- 一键采集该公众号的所有文章
- 显示采集进度条（已采集/总文章数）

### 2.3 文章管理
- 查看已采集的文章列表
- 支持按日期、标题搜索筛选文章
- 支持勾选文章进行批量导出
- 支持导入本地 HTML/JSON 文件

### 2.4 电子书导出
- 支持导出格式：EPUB、PDF、MOBI、DOCX
- 可自定义电子书封面、书名、作者
- 支持设置导出目录（章节顺序）
- 一键导出到指定目录

### 6.3 微信 API 调用策略

由于微信 API 需要登录态，直接从客户端调用会面临 CORS 和 Cookie 隔离问题。解决方案：

1. **方案 A：内嵌轻量服务器**
   - 在 Electron 主进程启动一个本地 Nitro 服务器
   - 渲染进程通过 `localhost` 调用 API
   - 解决 CORS 和 Cookie 共享问题

2. **方案 B：使用 Puppeteer/Playwright**
   - 通过无头浏览器控制微信网页版
   - 自动化完成登录和采集
   - 更加稳定但资源消耗较大

**推荐方案 A**，架构如下：

```
┌──────────────────────────────────────────────────────────────┐
│                     Electron 进程                             │

│  │  Main Process                                         │  │
│  │  ┌─────────────┐    ┌─────────────────────────────┐  │  │
│  │  │   Tray      │    │   Local Nitro Server         │  │  │
│  │  │   Menu      │    │   (端口: 38765)              │  │  │
│  │  │   DB        │    │                             │  │  │
```

### 6.4 数据存储

使用 `better-sqlite3` 在本地存储：

```sql
-- 账号表
CREATE TABLE accounts (
  id INTEGER PRIMARY KEY,
  name TEXT,
  cookie TEXT,
  token TEXT,
  created_at DATETIME,
  updated_at DATETIME
);

-- 文章表
CREATE TABLE articles (
  id INTEGER PRIMARY KEY,
  account_id INTEGER,
  title TEXT,
  link TEXT,
  content TEXT,
  author TEXT,
  publish_date DATETIME,
  cover_image TEXT,
  read_count INTEGER,
  like_count INTEGER,
  created_at DATETIME,
  FOREIGN KEY (account_id) REFERENCES accounts(id)
);

-- 采集任务表
CREATE TABLE tasks (
  id INTEGER PRIMARY KEY,
  account_id INTEGER,
  status TEXT, -- pending, running, completed, failed
  total INTEGER,
  current INTEGER,
  created_at DATETIME,
  updated_at DATETIME
);

-- 设置表
CREATE TABLE settings (
  key TEXT PRIMARY KEY,
  value TEXT
);
```
